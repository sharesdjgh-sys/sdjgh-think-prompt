# Think Prompt — Copy & 기능 정리

디자인 variant 제작 시 참고할 현재 앱의 모든 문구(copy)와 기능(feature)을 화면 단위로 정리합니다.

---

## 화면 구조 개요

```
앱 흐름: tutorial → input → (loading) → result
```

| 화면 | 컴포넌트 | 파일 |
|------|---------|------|
| 튜토리얼 (5단계) | `Tutorial` | `Tutorial.tsx` |
| 프롬프트 입력 | `App` (input step) | `App.tsx` |
| 분석 결과 | `AnalysisResult` | `AnalysisResult.tsx` |

## 4. API / 기능 정리

### 분석 기능 (`lib/analyzer.ts`)

- 모델: `claude-sonnet-4-6`
- 입력: 사용자가 입력한 프롬프트 문자열
- 출력: `ChatAnalysisResult` 객체

```ts
interface ChatAnalysisResult {
  scores: {
    clarity: number;   // 명확성 0~100
    context: number;   // 맥락 0~100
    scope: number;     // 범위 0~100
    format: number;    // 형식 지정 0~100
    role: number;      // 역할 부여 0~100
  };
  total: number;             // 5개 평균
  overall_comment: string;   // 전체 한 줄 평가
  feedback: {
    clarity: string;
    context: string;
    scope: string;
    format: string;
    role: string;
  };
  improved_prompt: string;          // 개선된 프롬프트
  changes: { what: string; why: string }[];  // 변경 내역
  tip: string;                      // 오늘의 팁
}
```

### 키보드 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+Enter` / `Cmd+Enter` | 프롬프트 분석 실행 |

### 상태 흐름

```
tutorial
  └─ onStart() → input
       ├─ handleAnalyze() → loading → result
       │     └─ onReset() → input
       └─ error → input (에러 메시지 표시)
```
